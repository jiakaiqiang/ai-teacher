## 👤 用户画像 1：运维工程师（主要用户）

### 基本信息

| 项目 | 内容 |
|------|------|
| **年龄** | 25-35 岁 |
| **工作年限** | 2-6 年 |
| **学历** | 本科/大专 |
| **常驻地** | 一线/新一线城市 |
| **薪资** | 15-30K/月 |

### 工作场景

**典型一天**
```
09:00  上班，查看监控大盘
09:30  处理昨夜告警工单（5-10 个）
10:00  例行巡检 20 台服务器
12:00  午休
14:00  处理突发告警（CPU 突增）
       ├ 登录多个监控系统
       ├ 查看 Grafana 指标
       ├ SSH 登录服务器查日志
       ├ 判断原因
       ├ 决定处理方式
       └ 创建工单 / 通知开发
       耗时：30-60 分钟
16:00  编写运维报告
18:00  下班（如无紧急告警）
```

**痛点场景**
```
痛点 1：告警泛滥
  - 每天收到 50+ 条告警
  - 90% 是噪音（CPU 短暂波动、网络抖动）
  - 真正的问题淹没在噪音中

痛点 2：多系统切换
  - Grafana 看指标
  - ELK 看日志
  - CMDB 查设备信息
  - Jira 创建工单
  - 来回切换，效率低

痛点 3：经验依赖
  - 新手不知道从哪查起
  - 相同问题每次都要重新分析
  - 老员工离职，经验流失

痛点 4：响应延迟
  - 异常发生到处置决策平均 30-60 分钟
  - 夜间告警响应更慢
  - 影响系统可用性
```

### 使用动机

**为什么会用这个产品？**
1. **减少告警噪音**：智能检测减少 90% 误报
2. **自动诊断**：Agent 30s 完成诊断，不用自己查
3. **一站式**：不用切换多个系统
4. **知识沉淀**：相似问题可以复用诊断

**期望价值**
- 每天节省 2-3 小时重复诊断时间
- 告警响应速度从 30 分钟 → 5 分钟
- 新手也能快速定位问题

### 使用场景

#### 场景 1：早晨查看大盘（每日必做）
```
行为：
  1. 打开监控大盘
  2. 查看昨夜异常列表
  3. 点击异常查看 Agent 诊断报告
  4. 决定是否需要人工介入

关注点：
  - 有多少个未处理异常？
  - 哪些是紧急的？
  - Agent 诊断是否准确？

期望：
  - 5 分钟内完成所有异常评估
  - 清晰的优先级排序
  - 一键批量标记为"已读"
```

#### 场景 2：处理突发告警（高频）
```
触发：收到钉钉/短信通知"srv-001 CPU 异常"

行为：
  1. 打开监控平台
  2. 查看该异常的诊断详情
  3. 查看 Agent 推荐的处理方案
  4. 如果认可，直接执行
  5. 如果不认可，手动处理并反馈

关注点：
  - Agent 诊断是否靠谱？
  - 推荐的处理方案是否可执行？
  - 历史上类似问题怎么解决的？

期望：
  - 80% 的异常可以直接采纳 Agent 建议
  - 20% 需要人工判断
  - 处理时间从 30 分钟 → 5 分钟
```

#### 场景 3：主动查询（低频）
```
行为：在 Chat 浮层输入
  "过去 1 周 srv-001 为什么频繁 CPU 告警？"

期望：
  - Agent 能分析趋势
  - 给出根因分析
  - 推荐长期解决方案（如扩容）
```

### 关注指标

| 指标 | 重要性 | 说明 |
|------|-------|------|
| 异常数量 | ⭐⭐⭐⭐⭐ | 今天有多少个异常？ |
| 诊断准确率 | ⭐⭐⭐⭐⭐ | Agent 诊断靠谱吗？ |
| 响应时间 | ⭐⭐⭐⭐ | 从异常到诊断多久？ |
| 误报率 | ⭐⭐⭐⭐ | 有多少是误报？ |
| 工单创建数 | ⭐⭐⭐ | 有多少需要人工处理？ |

### 典型抱怨

> "又是 CPU 告警，不知道是真有问题还是又是虚惊一场..."

> "每次都要去 Grafana、ELK、服务器来回看，太麻烦了"

> "老张知道怎么处理这种问题，但他离职了，现在没人知道..."

> "半夜被叫起来看告警，发现是虚惊，睡意全无..."

### 理想体验

> "早上打开看一眼，Agent 已经把昨夜的异常都分析好了，我只需要点'同意'就行"

> "异常发生后 1 分钟就收到诊断报告，直接按建议操作，不用自己查"

> "系统会记住每次处理经验，下次遇到类似问题自动参考"

## 👤 用户画像 2：SRE 工程师（次要用户）

### 基本信息

| 项目 | 内容 |
|------|------|
| **年龄** | 28-38 岁 |
| **工作年限** | 5-10 年 |
| **学历** | 本科/硕士 |
| **薪资** | 30-60K/月 |

### 工作场景

**职责**
- 系统稳定性负责人
- 制定监控策略
- 优化告警规则
- 编写运维自动化脚本
- 故障复盘

**痛点**
```
痛点 1：告警规则难调
  - 阈值太低 → 误报多
  - 阈值太高 → 漏报
  - 需要不断调整

痛点 2：故障复盘耗时
  - 每次故障需要人工收集日志
  - 写复盘报告 2-3 小时
  - 缺乏数据支撑

痛点 3：运维质量难量化
  - 响应时间、处理时间靠人工统计
  - 无法评估运维工程师工作量
```

### 使用场景

#### 场景 1：查看运维质量报表
```
行为：
  1. 查看本周异常统计
  2. 查看 Agent 诊断准确率
  3. 查看平均响应时间
  4. 导出报告给领导

关注点：
  - 系统可用性达标了吗？
  - 运维效率有提升吗？
  - Agent 替代了多少人工工作？
```

#### 场景 2：优化检测规则
```
行为：
  1. 查看误报率高的异常类型
  2. 调整检测阈值
  3. 观察调整后的效果

关注点：
  - 哪些检测规则需要优化？
  - 调整后误报率是否降低？
```

#### 场景 3：故障复盘
```
行为：
  1. 查看某次重大故障的完整诊断记录
  2. 查看 Agent 的工具调用链
  3. 分析根因
  4. 编写复盘报告

期望：
  - Agent 记录的诊断过程可作为复盘依据
  - 一键导出完整诊断数据
```

### 关注指标

| 指标 | 重要性 | 说明 |
|------|-------|------|
| MTTD（平均检测时间） | ⭐⭐⭐⭐⭐ | 异常发生到检测到多久？ |
| MTTR（平均恢复时间） | ⭐⭐⭐⭐⭐ | 从检测到恢复多久？ |
| Agent 准确率 | ⭐⭐⭐⭐ | 诊断靠谱吗？ |
| 误报率 | ⭐⭐⭐⭐ | 有多少告警是噪音？ |
| 系统可用性 | ⭐⭐⭐⭐⭐ | 99.5% 达标了吗？ |

### 理想体验

> "告警规则不用再手动调了，系统会基于历史数据自动优化"

> "故障复盘报告自动生成，我只需要补充结论"

> "一张图看清所有运维指标，给领导汇报有数据支撑"

## 👤 用户画像 3：技术管理者（辅助用户）

### 基本信息

| 项目 | 内容 |
|------|------|
| **年龄** | 32-45 岁 |
| **职位** | 技术经理 / CTO |
| **工作年限** | 8-15 年 |
| **薪资** | 50-100K/月 |

### 工作场景

**职责**
- 技术团队管理
- 系统稳定性把控
- 成本控制
- 向上汇报

**痛点**
```
痛点 1：运维成本高
  - 招 1 个运维工程师 20-30K/月
  - 需要 24 小时值班
  - 人力成本占比大

痛点 2：系统稳定性难量化
  - 领导问"系统稳定性怎么样？"
  - 只能凭感觉回答
  - 缺乏数据支撑

痛点 3：运维效率难评估
  - 不知道运维工程师每天做了多少工作
  - 无法评估 AI 带来的效率提升
```

### 使用场景

#### 场景 1：月度汇报
```
行为：
  1. 查看本月运维数据报表
  2. 导出数据
  3. 制作 PPT 向领导汇报

关注点：
  - 系统可用性达标了吗？
  - 故障次数比上月增加还是减少？
  - AI 带来了多少效率提升？
```

#### 场景 2：成本分析
```
行为：
  1. 查看 AI 调用成本
  2. 计算 AI 替代的人力成本
  3. 评估 ROI

关注点：
  - AI 每月花费多少钱？
  - 替代了多少人工时？
  - 值不值得投入？
```

### 关注指标

| 指标 | 重要性 | 说明 |
|------|-------|------|
| 系统可用性 | ⭐⭐⭐⭐⭐ | 99.5% 达标了吗？ |
| 故障次数 | ⭐⭐⭐⭐ | 本月比上月少了吗？ |
| 运维效率提升 | ⭐⭐⭐⭐ | AI 节省了多少人力？ |
| 成本 | ⭐⭐⭐ | AI 调用成本多少？ |

### 理想体验

> "一张图看清系统健康度，向领导汇报有底气"

> "AI 每月只花 500 元，但节省了 2 个人力月，ROI 很高"

## 📊 用户需求优先级矩阵

```
高价值 ─────────────────────────
  │   
  │   [智能检测]    [自动诊断]
  │      P0            P0
  │   
  │   [知识沉淀]    [数据报表]
  │      P1            P1
  │   
低价值 ─────────────────────────
     低频              高频
          使用频率 →
```

**P0（必须有）**
- 智能异常检测
- Agent 自主诊断
- 实时监控大盘
- 异常列表 + 详情

**P1（应该有）**
- 知识沉淀与复用
- Chat 主动查询
- 运维数据报表

**P2（可以有）**
- 配置中心
- 多租户隔离
- 移动端适配

## 🎯 产品设计策略

### 针对运维工程师（70%）
1. **降低学习成本**：UI 简洁直观，5 分钟上手
2. **减少操作步骤**：80% 场景 3 次点击内完成
3. **建立信任**：诊断过程透明，显示 Agent 思考过程
4. **及时反馈**：异常 → 诊断 → 展示全程 < 1 分钟

### 针对 SRE 工程师（20%）
1. **提供深度数据**：完整的诊断记录、工具调用链
2. **支持优化**：可调整检测规则、可导出数据
3. **开放 API**：支持集成到现有运维系统

### 针对技术管理者（10%）
1. **一页看全局**：Dashboard 显示核心指标
2. **支持导出**：数据可导出为 Excel/PDF
3. **成本透明**：AI 调用成本清晰展示
