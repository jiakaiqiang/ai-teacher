```

  ✅ LangChain / LangGraph 概念已掌握
```

## ⚠️ 关键里程碑与风险点

### 里程碑检查点

| 时间点 | 检查项 | 不达标风险 |
|-------|--------|-----------|
| Day 60 | 4 个项目全部上线 + 简历完善 | 第一次跳槽推迟 |
| Day 120 | LangChain.js 深度项目 1 个 | 能力深度不足 |
| Day 180 | 第一次跳槽成功 OR 有 3+ 面试 | 需调整策略 |
| Day 300 | LangGraph 生产级项目 2 个 | 第二次跳槽困难 |
| Day 420 | Python 基础 + FastAPI 项目 1 个 | Python 转型失败 |
| Day 540 | 双栈项目至少 2 个 | 双栈标签立不住 |

### 风险应对

**风险 1：Day 60 时 4 个项目未完成**
- 应对：延长到 Day 75，但必须完成
- 代价：第一次跳槽窗口推迟

**风险 2：Day 180 跳槽失败**
- 应对：继续深耕到 Day 240，增加项目深度
- 调整：第二次跳槽提前准备，不等 Day 365

**风险 3：Python 学习进度慢**
- 应对：Python 基础用 AI 编程暴力加速
- 策略：只学必要的，不追求 Python 专家
