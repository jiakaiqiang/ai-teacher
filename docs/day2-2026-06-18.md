## 📚 学（知识点速查）

### 1. <script setup> 语法

- **是什么**：组件顶层写逻辑的语法糖，里面声明的变量、函数、import 的组件，模板里直接能用，无需 return。
- **何时用**：写所有 Vue 3 组件的默认写法，比 setup() 函数更简洁。
- **易错点**：<script setup> 里没有 this，别用选项式 API 的写法（如 this.xxx）。
- **对比 React**：类似函数组件体，但它只执行一次（不像 React 函数每次渲染都重跑），响应性靠 ref/reactive 而非重新执行。

### 2. ref vs reactive

- **是什么**：ref 包装任意值（含基本类型），用 .value 读写；reactive 只接对象/数组，直接读写属性。
- **何时用**：基本类型、需要整体替换的值用 ref；一组相关属性的对象用 reactive。拿不准就统一用 ref。
- **易错点**：reactive 对象解构后丢失响应性；ref 在 <script> 里要写 .value，模板里会自动解包不用写。
- **对比 React**：ref 像 useState，但可直接改 .value 不需 setState；reactive 没有 React 对应物，更接近可变的响应对象。

### 3. computed 计算属性

- **是什么**：基于已有响应式数据派生出新值，依赖变了才重新计算，结果有缓存。
- **何时用**：某个值是由其它状态算出来的（如总价 = 单价×数量）时用它，而不是写函数每次调用。
- **易错点**：computed 默认只读，别在里面改数据；依赖必须是响应式的，否则不会更新。
- **对比 React**：相当于 useMemo，但依赖自动收集，不用手写依赖数组。

### 4. props / emit 父子通信

- **是什么**：props 是父传子的只读数据（defineProps），emit 是子通知父的事件（defineEmits）。
- **何时用**：拆组件时：父往子塞数据用 props，子要把动作/数据反馈给父用 emit。
- **易错点**：props 是只读的，子组件不能直接改 prop；要改就 emit 事件让父去改（单向数据流）。
- **对比 React**：props 一样；emit 相当于父传给子的回调函数 prop，只是 Vue 用事件机制表达。

## 🧠 自测（学完先别看答案，自己答一遍）

> 答不上来就回到「📚 学」对应小节再过一遍，再动手做。

1. 为什么 props 是只读的？子组件想改怎么办？
   <details><summary>点开看答案</summary>

   为保证单向数据流，数据源唯一好追踪。子组件需要改时 emit 一个事件，由父组件去修改后通过 props 流回来。
   </details>

2. reactive 包的对象解构出来后为什么不更新了？ref 有这个问题吗？
   <details><summary>点开看答案</summary>

   reactive 解构会拿到普通值，脱离响应式代理所以不更新（可用 toRefs 解决）。ref 通过 .value 持有引用，不存在解构丢响应的问题。
   </details>

3. computed 和普通函数都能算出派生值，区别在哪？
   <details><summary>点开看答案</summary>

   computed 有缓存，依赖不变时多次访问不重算；普通函数每次调用都执行。展示派生值优先用 computed。
   </details>
